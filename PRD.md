**작성일**: 2026-02-06
**작성자**: 코드팩토리 PM팀
**버전**: v1.0
**상태**: Draft

---

## 1. 개요 (Executive Summary)

본 문서는 오프라인 매장을 운영하는 소상공인이 쉽고 빠르게 온라인 판매를 시작할 수 있도록 지원하는 **올인원 모바일 쇼핑몰 솔루션**의 제품 요구사항을 정의합니다.

### 제품 비전
> "스마트폰 하나로, 1시간 안에 나만의 온라인 스토어를 연다"

---

## 2. 문제 정의 (Problem Statement)

### 2.1 현재 시장 상황
- 오프라인 매장 운영 소상공인 중 **약 70%**가 온라인 판매 의향 보유
- 그러나 실제 온라인 진출 비율은 현저히 낮음

### 2.2 기존 솔루션의 한계

| 문제점 | 상세 내용 |
|--------|----------|
| **기능 과잉** | 평균 30개 이상의 불필요한 기능으로 인한 복잡한 UI/UX |
| **높은 수수료** | 거래액 대비 10%에 달하는 플랫폼 수수료 |
| **높은 진입 장벽** | 웹사이트 제작, 결제 연동 등 기술적 허들 |
| **PC 중심 설계** | 매장에서 즉시 사용하기 어려운 데스크톱 기반 인터페이스 |

---

## 3. 타겟 사용자 (Target Users)

### 3.1 핵심 페르소나

**김사장님 (38세, 의류 편집샵 운영)**

- **배경**: 홍대 인근에서 5년째 편집샵 운영
- **디지털 역량**: 인스타그램 활발히 사용, 팔로워 2,000명 보유
- **Pain Point**:
  - DM으로 주문받고 계좌이체로 결제받는 비효율적인 프로세스
  - 쇼핑몰 플랫폼 가입했으나 복잡해서 포기
- **니즈**: 신상품 입고 시 바로 사진 찍어 올리고 판매하고 싶음

### 3.2 타겟 세그먼트

| 구분 | 특성 |
|------|------|
| **연령대** | 30~40대 |
| **업종** | 의류, 잡화, F&B, 핸드메이드 등 개성 있는 상품 판매 |
| **디지털 리터러시** | SNS 활용 능숙 / 코딩·웹개발 경험 無 |
| **운영 형태** | 오프라인 매장 기반, 온라인 확장 희망 |

## 4. 사용 시나리오 (Use Cases)

### 시나리오 1: 신상품 즉시 등록
```
1. 매장에 신상품 입고
2. 스마트폰으로 상품 사진 촬영
3. 앱에서 사진 선택 → 상품명/가격 입력
4. '등록' 버튼 탭 → 즉시 스토어에 노출
```

### 시나리오 2: 주문 접수 및 처리
```
1. 고객이 스토어에서 상품 주문 & 결제
2. 사장님 스마트폰에 푸시 알림 수신
3. 앱에서 주문 상세 확인
4. 상품 포장 후 '배송 시작' 버튼 탭
5. 고객에게 배송 알림 자동 발송
```

### 시나리오 3: 매출 확인
```
1. 앱 홈 화면에서 오늘/이번 주/이번 달 매출 한눈에 확인
2. 베스트셀러 상품 및 재고 현황 파악
3. 정산 예정 금액 확인
```

## 5. 제안 솔루션 (Proposed Solution)

### 5.1 제품 컨셉

**"손안의 쇼핑몰"** - 스마트폰 앱 하나로 상품 등록부터 주문 관리, 결제, 정산까지 모든 것을 해결하는 올인원 모바일 커머스 플랫폼

### 5.2 핵심 가치 제안 (Value Proposition)

| 기존 플랫폼 | 우리 솔루션 |
|------------|------------|
| 30개+ 복잡한 기능 | **5개 핵심 기능**에 집중 |
| 10% 높은 수수료 | **업계 최저 수수료** |
| PC 기반 관리자 페이지 | **모바일 퍼스트** 설계 |
| 스토어 개설까지 수일 소요 | **1시간 내 오픈** 가능 |

### 5.3 핵심 기능 (Core Features)

#### 1) 스토어 개설
- 상호명, 로고, 소개글만 입력하면 즉시 스토어 URL 생성
- 기본 테마 제공 (커스터마이징 불필요)

#### 2) 상품 관리
- 사진 촬영 → 상품명/가격 입력 → 등록 (3단계)
- AI 기반 상품 설명 자동 생성 (선택)
- 재고 관리 및 품절 자동 처리

#### 3) 주문 관리
- 실시간 푸시 알림
- 주문 상태 원탭 변경 (신규 → 준비중 → 배송중 → 완료)
- 송장번호 입력 시 고객 자동 알림

#### 4) 결제 시스템
- 토스페이먼츠 연동 (카드, 간편결제)
- 별도 사업자 심사 없이 즉시 결제 수신 가능
- 주 단위 자동 정산

#### 5) 대시보드
- 오늘의 매출/주문 현황
- 베스트셀러 TOP 5
- 정산 예정 금액

## 6. 성공 지표 (Success Metrics)

### 6.1 핵심 목표
> **소상공인의 온라인 커머스 진입 장벽을 낮춘다**

### 6.2 KPI

| 지표 | 목표 | 측정 기간 |
|------|------|----------|
| **활성 스토어 수** | 500개 | 론칭 후 6개월 |
| **누적 거래액** | 10억원 | 론칭 후 12개월 |
| **스토어 개설 완료율** | 80% | 가입 후 24시간 내 |
| **첫 판매까지 소요 시간** | 7일 이내 | 스토어 개설 후 |
| **월간 재사용률 (MAU/가입자)** | 60% | 월간 |

### 6.3 품질 지표

| 지표 | 목표 |
|------|------|
| 앱 스토어 평점 | 4.5점 이상 |
| 고객 문의 응답 시간 | 24시간 이내 |
| 결제 성공률 | 99% 이상 |

## 7. 범위 정의 (Scope)

### 7.1 MVP 포함 (In Scope)

- iOS / Android 네이티브 앱
- 웹 기반 고객용 스토어 페이지
- 상품 등록/수정/삭제
- 주문 접수 및 상태 관리
- 토스페이먼츠 결제 연동
- 기본 대시보드

### 7.2 MVP 제외 (Out of Scope)

- PC용 관리자 페이지
- 마케팅 도구 (쿠폰, 할인)
- 고객 리뷰 시스템
- 다국어 지원
- POS 연동
